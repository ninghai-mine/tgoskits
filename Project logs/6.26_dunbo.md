# 6.26 工作日志

## GPA 修正

**根因**: `KERNEL_IMAGE_TEXT_PA` 漏了 `kernel_load_addr` 的 2MB TEXT_OFFSET。

- 正确 HPA = VA - 0xffff800080000000 + 0x223600000 + TEXT_OFFSET(0x200000)
- 修复 `log.rs`: `KERNEL_IMAGE_TEXT_PA`, `DEFAULT_LOG_BUF_GPA`, `PRINTK_RB_STATIC_GPA`
- 修复 `snapshot.rs`: kallsyms 各数组 GPA + `collect_kernel_log` 调用参数
- 修复 `vmlinux.symtab`: 相对地址存储（match `SymbolTable::lookup()` 的 `addr.wrapping_sub(kernel_base)` 逻辑）
- 修复 `gen-kallsyms-config.sh`: 同步 GPA 公式

## 崩溃类型扩展

`crash_test.c` 从 3 种扩展到 8 种：

| action | 方式 | 硬件异常 |
|--------|------|----------|
| null | NULL 读 | Data Abort |
| write_null | NULL 写 | Data Abort (write) |
| bug | BUG() | BRK → Oops |
| undefined | .inst 0x0 | UDF → Oops |
| double_free | 双重 kfree + BUG() | BUG |
| stack_overflow | 递归爆栈 | 栈越界 Data Abort |
| exec_null | 跳转 NULL | Instruction Abort |
| corrupt_mem | 随机写 | Data Abort (perm fault) |

## Crash Info 增强

- `analyzer.rs`: 新增 `esr_el1`, `far_el1`, `crash_detail` 字段
- `decode_esr()`: ESR_EL1 硬件异常解码（Data Abort/Instruction Abort/Undefined/BUG）
- `extract_panic_reason()`: 从 dmesg 提取 panic 原因（SysRq/NULL/BUG/Undefined/stack overflow）
- `panic_reason_has_hw_fault()`: 判断 ESR=0 时是否仍有硬件异常（内核 EL1 消化了）
- `clean_panic_reason()`: 清理 prb ringbuffer 造成的尾随垃圾字符
- `dedup_backtrace()`: 栈溢出时相同递归帧合并显示
- `console.rs` cmd_info: 显示 ESR_EL1/FAR_EL1/Detail

## 动态 GPA 框架 (`capture/locate.rs`)

- 集中管理内核符号 VA offset + HPA_BASE
- `offset_to_hpa(offset)`: 统一 VA→HPA 转换
- 当前使用硬编码 fallback（指纹扫描因 alternatives patching 移除）
- 队友换内核后只需更新 `locate::offsets` 模块常量

## 模块编译修复

- 关闭 `CONFIG_MODVERSIONS`（外部模块不需要 symvers）
- `crash_test.ko` strip debug sections（修复 "Invalid ELF section header overflow"）
- 构建命令: `KBUILD_MODPOST_WARN=1 make ... && aarch64-linux-gnu-strip --strip-debug`
